# Deck 3 — Collection Theme Cards (×10)

**ISO/IEC 29119-3 mapping:** Test Design Specification
**ISO fields preserved on every card:** *Feature set ID · Test conditions · Traceability to requirements · Coverage criteria*
**Function in play:** Groups Outfits under a shared design intent. Every Outfit on your mat must satisfy the active Theme.

---

## CT-01 — "Mediterranean Breeze"

- *Feature set ID:* MB-SS26
- *Test conditions:* Light fabrics; sand / coral / off-white palette; flowing silhouettes
- *Traceability:* REQ-FASHION-MB-01..03
- *Coverage:* Day, transitional, evening — at least one per Outfit set

## CT-02 — "Concrete Geometry"

- *Feature set ID:* CG-AW26
- *Test conditions:* Structured tailoring; grey/black palette; sharp angles
- *Traceability:* REQ-FASHION-CG-01..03
- *Coverage:* Architectural silhouettes

---

## Why these matter (educational intent)

A Test Design Specification is the **most under-taught document** in 29119-3. Students often jump straight from a "plan" to writing test cases. By forcing players to place a Theme between the Roadmap and the Outfits, the game teaches that test cases are not free-floating — they belong to a designed feature set with explicit coverage criteria and explicit traceability to requirements.
