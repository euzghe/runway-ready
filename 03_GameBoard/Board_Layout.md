# Board Layout

*Runway Ready* uses **per-player Studio Mats** plus a **central Common Area**. There is no single shared board because every player is preparing their own show — the visual separation reinforces that each player owns their own documentation.

## The Studio Mat (one per player)

Suggested size: A4 landscape. Five labelled zones, top-to-bottom matching the documentation hierarchy.

```
┌────────────────────────────────────────────────────────────────┐
│  STUDIO MAT — Player Name: ________________________            │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  ZONE 1 — BRAND IDENTITY        (Organizational Test     │  │
│  │  [ slot for 1 card ]              Strategy)              │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  ZONE 2 — RUNWAY ROADMAP        (Project Test Plan)      │  │
│  │  [ slot for 1 card ]                                     │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  ZONE 3 — COLLECTION THEME      (Test Design Spec)       │  │
│  │  [ slot for 1 card ]                                     │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                │
│  ┌─────────────┬─────────────┬─────────────┬───────────────┐  │
│  │  ZONE 4 — OUTFIT DETAILS    (Test Case Specs)            │  │
│  │  [Slot 1]   │  [Slot 2]   │  [Slot 3]   │  [+1 / +2]    │  │
│  └─────────────┴─────────────┴─────────────┴───────────────┘  │
│                                                                │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  ZONE 5 — RUNWAY INSTRUCTIONS   (Test Procedure Spec)    │  │
│  │  [ slot for 1 card ]                                     │  │
│  └──────────────────────────────────────────────────────────┘  │
│                                                                │
│  Rehearsal Summary pad ↓ (paper, fillable)                     │
│  ┌──────────────────────────────────────────────────────────┐  │
│  │  Outfit ID │ Pass/Fail │ Originating Doc Field           │  │
│  │  ─────────────────────────────────────────────────────── │  │
│  │            │           │                                 │  │
│  │            │           │                                 │  │
│  │            │           │                                 │  │
│  └──────────────────────────────────────────────────────────┘  │
└────────────────────────────────────────────────────────────────┘
```

Why this matters: the **vertical order of zones is the documentation hierarchy**. A player physically cannot place an Outfit above the Theme — there is no slot. Spatial constraint reinforces structural constraint.


## Optional: the "Runway" zone (theatrical variant)

For groups who want a theatrical twist, designate a strip of table between players as **the Runway**. During Rehearsal, physically move Outfit cards from your mat onto the runway, in the order declared by your Runway Instructions card. Failed Outfits are flipped face-down on the runway — a visual reminder of which test cases failed and from where.

This is optional. The core game requires nothing more than the Studio Mats and the Common Area.
